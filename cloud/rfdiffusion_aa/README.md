# RFdiffusionAA cloud package — de-novo pocket around the L4F anchor

Self-contained instructions to run RFdiffusionAA on a **≥16 GB GPU cloud host**
(the local 8 GB box is deliberately not used). Goal: diffuse a protein backbone
**around the corrected fluoroaromatic anchor** so the **pocket shape** can
complement the para-fluorine — the discriminator the LigandMPNN dry-run showed
sequence design cannot provide.

## What's baked in

`input_L4F_pocket.pdb` — the cloud input, generated by
`src/prep_rfdiffusion_aa_input.py`. It contains:

- **Carrier protein** = scaffold chain A (172 residues). Required only so
  RFdiffusionAA's `process_target()` can build the contig frame from CA atoms;
  with the de-novo contig below it is **parsed then discarded** (`insert_contig`
  keeps only the ligand as motif) — exactly the upstream 7v11/OQO pattern.
- **Ligand** = the corrected **L4F anchor**, resname **`PFF`**, HETATM 1392–1400:
  `CA CB CG CD1 CD2 CE1 CE2 CZ F` — **8 carbons + the single para-F, zero oxygens**,
  with CONECT. This is byte-identical chemistry to the anchor-fix GATE deliverable
  `data/raw/ligand_L4F.pdb`.

**Verification stamp (confirmed before packaging):**
```
anchor self-check : C–F 1.339 Å · CG···CZ(para) 2.772 Å · F nearest ring C = CZ (PARA) · O count = 0
cloud input check : resname PFF · 9 heavy atoms · F count = 1 · O count = 0  → matches the GATE ligand
```
RFdiffusionAA reads the ligand via OpenBabel (`parse_mol`): atom identity from the
**element column**, bonds from **CONECT/geometry**, **no CCD/template lookup** — so
`PFF` is just a matching label and the para-F is seen as fluorine. Nothing to edit.

## One-time setup on the cloud host (~13 GB + Apptainer)

```bash
git clone --recurse-submodules https://github.com/baker-laboratory/rf_diffusion_all_atom.git
cd rf_diffusion_all_atom
# container (~11.8 GB) and weights (~1.27 GB), both into this repo dir:
wget http://files.ipd.uw.edu/pub/RF-All-Atom/containers/rf_se3_diffusion.sif
wget http://files.ipd.uw.edu/pub/RF-All-Atom/weights/RFDiffusionAA_paper_weights.pt
# Apptainer: https://apptainer.org/docs/admin/main/installation.html
```

## Run

Copy `input_L4F_pocket.pdb` into the repo's `input/` dir, then run from the repo dir:

```bash
mkdir -p input output/l4f_pocket
cp /path/to/input_L4F_pocket.pdb input/
apptainer run --nv rf_se3_diffusion.sif -u run_inference.py \
  inference.deterministic=True \
  diffuser.T=100 \
  inference.output_prefix=output/l4f_pocket/des \
  inference.input_pdb=input/input_L4F_pocket.pdb \
  inference.ligand=PFF \
  contigmap.contigs=[\'120-120\'] \
  inference.num_designs=1 \
  inference.design_startnum=0
```

(Equivalent repo-integrated runner: `src/run_rfdiffusion_aa.sh`, same parameters.)

### Tunables
- `contigmap.contigs=[\'N-N\']` — pocket length. Start ~**120** for this small
  aromatic; 80–150 is sane. Smaller = more compact (less "pool-noodle" risk).
- `inference.num_designs` — raise to fan out once the first design looks sane.
- `diffuser.T` — 100 (README ligand example) … 200 (more steps).

### Expected outputs
- `output/l4f_pocket/des_0.pdb` — the design (backbone; sequence is meaningless
  until LigandMPNN is run).
- `des_0_Xt-1_traj.pdb`, `des_0_X0-1_traj.pdb` — trajectories.

## After the backbone exists (the actual discrimination test)
1. **LigandMPNN** sequence on the new pocket (env already installed locally).
2. **AF2 multimer co-fold** with the validated instrument
   (`src/run_multimer.sh` → `src/parse_interface.py`).
3. **Physics affinity gap**: score **L4F vs PHE** in the designed pocket — the
   F-absent control is `data/raw/ligand_PHE.pdb` (identical minus the para-F).
   The *gap* (not raw affinity) is the win condition.
